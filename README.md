Інструкція користувача
1. Загальні відомості 
– Повна назва програмного продукту: Багаторівнева система автоматизації складського обліку «MiniWarehouse». 
– Версія: 1.0. 
– Автор: Макаренко Роман Андрійович, група ТЦР-22. 
– Платформа реалізації: мова C#, фреймворк ASP.NET Core 9.0. 
– Тип застосунку: Web API (з інтерфейсом Swagger та Razor Pages).
2. Функціональне призначення 
Програмний продукт призначений для автоматизації процесів обліку товарів, управління залишками на складах та обробки замовлень. Система розрахована на використання адміністраторами складу (для ведення номенклатури) та менеджерами (для перевірки залишків і створення замовлень). Основні функції включають: перегляд списку товарів, пошук товару за ідентифікатором, додавання нових позицій (із захистом JWT) та автоматичне списання залишків при замовленні.
3. Умови застосування програми 
Для коректної роботи програмного продукту необхідні наступні умови: 
– Тип пристрою: ПК або ноутбук. 
– Операційна система: Windows 10/11, Linux або macOS. 
– Мінімальні характеристики: процесор 2 ГГц, 4 ГБ оперативної пам’яті, 500 МБ вільного місця на диску. 
– Необхідне програмне забезпечення: встановлений .NET 9.0 SDK, середовище розробки Visual Studio 2022 (для запуску з вихідного коду), встановлений сервер баз даних Microsoft SQL Server LocalDB.
4. Інструкція із запуску 
Запуск проєкту з вихідного коду здійснюється наступним чином:
Відкрити файл рішення MiniWarehouse.sln у середовищі Visual Studio 2022.
Переконатися, що у якості стартового проєкту вибрано MiniWarehouse.Web.
Дочекатися автоматичного відновлення залежностей (NuGet Packages).
У консолі диспетчера пакетів (Package Manager Console) виконати команду Update-Database для створення структури бази даних.
Натиснути кнопку запуску (F5 або Ctrl+F5). Програма автоматично відкриє браузер зі сторінкою Swagger UI (за адресою https://localhost:7169/swagger).
5. Повідомлення користувачу 
При роботі з системою можуть виникати наступні типи повідомлень: 
– Повідомлення про помилку 401 (Unauthorized):
Причина виникнення: спроба додати товар без попередньої авторизації.
Рекомендовані дії користувача: перейти до методу POST /api/Auth/login, отримати JWT-токен і ввести його в меню "Authorize". 
– Повідомлення про помилку 400 (Bad Request):
Причина виникнення: спроба замовити товару більше, ніж є на складі, або передача некоректних даних у форматі JSON.
Рекомендовані дії користувача: перевірити наявну кількість товару (через метод GET) та правильність заповнення всіх полів (наприклад, ціна має бути числом). 
– Інформаційне повідомлення 201 (Created):
Причина виникнення: товар успішно додано до бази даних.
6. Опис роботи програми Робота із системою здійснюється через графічний інтерфейс Swagger UI або безпосередньо через браузер (Razor Pages). Сценарій додавання товару (доступний тільки для адміністратора):
Відкрити метод POST /api/Auth/login. У тілі запиту передати логін admin та пароль admin123.
Скопіювати отриманий токен.
Натиснути кнопку "Authorize" у верхній частині екрана, ввести слово "Bearer", поставити пробіл і вставити скопійований токен. Натиснути кнопку підтвердження.
Відкрити метод POST /api/Products.
У тіло запиту (Request Body) вставити JSON-об'єкт із даними нового товару (SKU, назва, кількість, ціна, ідентифікатор складу).
Натиснути "Execute". Система перевірить токен, валідує дані та додасть товар до бази.
Сценарій створення замовлення:
Відкрити метод POST /api/Products/order.
У тілі запиту передати ідентифікатор товару (ProductId) та кількість для замовлення (ProductQuantity).
Натиснути "Execute". Якщо товару достатньо, система автоматично зменшить його залишок на складі та поверне підтвердження.
